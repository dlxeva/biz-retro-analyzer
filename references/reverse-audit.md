# Reverse Audit

Use this file when the current story feels too smooth, too confident, or too dependent on one narrator.

## Purpose

The reverse audit is not for contrarian style points.  
It is for stress-testing whether the current interpretation is actually supported by the evidence.

## Run a Reverse Audit When

- the user strongly agrees with the current narrative
- the meeting outcome seems "obvious"
- one party's framing dominates the record
- pricing, role, or leverage judgments are becoming important
- the product direction is being treated as settled too early

## Questions to Ask

### 1. What is most likely overstated?

- Did we overstate how much they need us?
- Did we overstate how settled the direction is?
- Did we overstate our pricing leverage?

### 2. What is most likely understated?

- Did we understate the role of another participant?
- Did we understate missing evidence about execution or acceptance?
- Did we understate how much of the story is still tactical?

### 3. What is inferred but not yet proven?

- role motives
- implementation boundaries
- project longevity
- actual willingness to pay

### 4. What would falsify the current story?

- another meeting with different participants
- a conflicting transcript
- a missing approval step
- a different interpretation of the same "trial scenario"

## Output Form

When you run reverse audit, produce at least:

- 3 judgments that still look sound
- 3 judgments that may be too confident
- 3 assumptions that should remain explicit
