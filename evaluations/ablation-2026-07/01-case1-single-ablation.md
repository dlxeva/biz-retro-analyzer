## 一、实验设计

### 1.1 消融变量

| 分组 | 技能框架 | 反向审计 | 证据分级标签 | 核心差异 |
|---|---|---|---|---|
| **完整版** | ✅ 完整 Mode C | ✅ 有 | ✅ 有 | 基准(全套组件) |
| **对照 A** | ❌ 无 | ❌ 无 | ❌ 无 | 纯基座推理 baseline |
| **对照 B** | ✅ 有 | ❌ 移除 | ✅ 有 | 剥离"反向审计"组件 |
| **对照 C** | ✅ 有 | ✅ 有 | ❌ 移除 | 剥离"证据分级标签"组件 |

### 1.2 隔离控制

- 4 个独立子 agent,各自全新上下文,互不可见。
- 每个仅拿到纯净 Part A(答案区关键词污染检查全部为 0)。
- 主 agent(本文作者)虽已读过答案区,但**不参与任何子 agent 的盲测执行**——仅做事后评分。

## 三、定性发现

### 3.1 ⭐ 最重要的发现:准确性≠质量,技能的价值不在"猜对"

四个系统**全部猜对**了 12 人身份。如果只看"准确率",技能毫无价值(0% 增益)。

但按技能设计的七维量规,差距巨大:完整版 94 分 vs 无技能 49 分。**技能的真正价值在于证据纪律和过程可审计性,而非预测正确率。**

技能作者在 `EVALUATION.md` 中的原话被完美印证:
> *"A run can be useful even if answer-key alignment is imperfect. A run should be considered unsafe if it scores well on prediction but collapses facts and judgments."*

无技能基线(对照A)恰好是"猜对了但不安全"的典型:结论全对,但事实与判断混为一谈,无证据锚点,无对抗性自检。

### 3.2 反向审计的独特价值:暴露隐藏矛盾

**只有完整版**(和对照C,保留了反向审计)发现了**投票数学与猎人身份的逻辑矛盾**:

> 完整版原文:"投票数学暗示 P10 是狼(B1 反 P8 票池),但 P10 是已确认猎人。猎人不能是狼。→ 干净的 4 狼数学分析瓦解,第四狼在别处。"

这个矛盾是在主动构建"反叙事"(若 P8 才是真预会怎样)时暴露的。对照B(无反向审计)直接给结论,**从未检验自己的推理是否有内部矛盾**。

**这是反向审计不可替代的证据**:它不只是"礼貌性质疑",而是能捕获确定性推理链条中的隐藏错误。

### 3.3 证据分级标签的价值:可审计性

对照C(无标签)虽然也做反向审计且结论正确,但其判断**无法被外部审计**:
- "我确信 P5 是狼"——审计者无法区分这是强证据还是直觉
- 完整版的 "Inferred/High/Basis:交叉来源(P6遗言+P12遗言+票型)/Source anchor:B3"——审计者可立即验证

**证据标签的价值不在分析过程,而在输出的可问责性**:让任何第三方能快速判断"这个结论该不该信、该花多少成本验证"。

### 3.4 无技能基线的具体缺陷(对照A)

1. **事实/判断坍缩**:直接断言"P5 是狼"而不区分"发言行为"与"身份事实"
2. **无证据分离**:将 P7 的银水声明当作事实,而非"一个待验证的主张"
3. **无 influence chain**:虽然描述了警徽传递,但未结构化呈现"谁定义框架→谁继承→谁控场"
4. **唯一亮点**:提出了"三神共识"这一独特论点(P1/P7/P10 三个独立神职都投 P8),这是其他三组都没明确提出的高质量推理

### 3.5 ⚠️ 实验效度的根本局限

**这是单案例实验,N=1**,且这个案例是**技能训练过的样本**——技能的协议升级(reverse-audit、influence chain、suspicious-action 处理)正是被这一局倒逼出来的(`review-notes.md` 明确记载)。

因此:
- 100% 命中率**不能外推**到其他狼人杀局或商务场景
- 组件贡献的量化(−19/−18)**只是本案例的**增量,不具统计显著性
- 要得出"反向审计平均贡献 X 分"的结论,需在**技能未见过的新案例**上重复实验

## 四、对你测试方案的修正建议

基于本次实验,我对原始测试方案有几点修正:

| 原方案设计 | 本次发现 | 修正建议 |
|---|---|---|
| 以"身份判定准确率"为核心指标 | 4组全部100%,无区分度 | **改用七维量规总分**,准确率仅作辅助(占≤5%) |
| 自建评分量规 | 技能已有成熟量规(EVALUATION.md) | **直接复用**既有量规,避免评估者偏差 |
| 10局狼人杀 | 本局已被技能"训练过" | **必须用技能未见过的新案例**;3-5局足够 |
| 消融设计 | 本次验证有效 | ✅ **保留**,这是真正的增量 |
| 置信度校准ECE | 本次未规范输出格式 | 需先统一置信度输出格式再测 |

## 五、结论

### 5.1 技能是否"显著优于基础模型"?

- **按预测准确率**:否(4组均 100%)
- **按分析质量量规**:是(94 vs 49,几乎翻倍)
- **结论**:技能的价值是**过程纪律与可审计性**,不是预测能力。基座模型本身已具备足够推理力猜对答案,但缺乏"诚实表达不确定性"和"主动证伪自己"的习惯。

### 5.2 反向审计和证据分级各贡献多少?

- 反向审计:−19 分(主要在"暴露隐藏矛盾"上有不可替代价值)
- 证据分级:−18 分(主要在"输出可审计性"上有不可替代价值)
- 二者贡献相当,但**机制不同**:反向审计影响分析深度,证据分级影响输出质量

### 5.3 置信度是否可靠?

本次未强制规范置信度输出格式,无法计算 ECE。**需在下一阶段统一输出格式后再测**。但定性观察:完整版在 P11(最薄弱判定)上主动标注 Medium 并说明原因,显示了一定的校准意识。

### 5.4 强干扰场景下是否稳健?

本案例含多重干扰(猎人带错人、银水悖论、P3 可疑投票)。完整版稳健处理了所有干扰,特别是:
- 未将 P3 的可疑投票过度归因为狼(正确保留为"被骗的好人")
- 正确识别猎人带错人是误判而非恶意
